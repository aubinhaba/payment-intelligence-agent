# pia-api

REST Backend-For-Frontend (BFF) consumed exclusively by the Angular dashboard.

## Responsibility

Thin HTTP faĂ§ade over the application use cases and read-side repositories:

| Resource            | Endpoint                   | Use case / query         |
|---------------------|----------------------------|--------------------------|
| Transactions        | `GET /api/transactions`    | Paginated read           |
| Anomalies           | `GET /api/anomalies`       | Filterable read          |
| Anomaly detail      | `GET /api/anomalies/{id}`  | Single anomaly + report  |
| Reports             | `GET /api/reports`         | Paginated metadata       |
| Report content      | `GET /api/reports/{id}`    | Markdown payload from S3 |
| Metrics summary     | `GET /api/metrics/summary` | KPI aggregates           |

## Rules

- Depends on `pia-application` only (no direct AWS SDK usage).
- All request DTOs validated via `jakarta.validation` (`@Valid`).
- Consistent error contract via `GlobalExceptionHandler`.
- OpenAPI 3.1 document generated by `springdoc` at `/v3/api-docs`.
- Strict CORS: only the dashboard origin is allowed.
